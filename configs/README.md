# Config Registry

This directory contains Hydra experiment configs. Treat configs as part of the
public API: scripts should select a config by name and pass only small,
explicit overrides for dataset paths, checkpoints, batch size, and GPUs.

Workspace selection is route-specific:

```yaml
_target_: src.workspace.ActionHiddenWMWorkspace
```

Each config points directly at the workspace implementation for that route.

Canonical goal checkpoint / action-hidden pairing for the next clean run:

```text
VLA base:
  data/ckpts/VLA_model_256/libero_goal

pi0 VLA action head / encoder state:
  data/ckpts/pi0_query_vla_libero_goal/epoch003_train_vla_loss1.255_success8of10.ckpt

Precomputed action-hidden sidecar:
  data/processed_data/libero_goal_no_noops_t_256_pi0_action_hidden_vla_policy_h2

Action horizon:
  5

Action head type:
  pi0_query
```

Do not mix this with pooled Rynn hidden sidecars. The action-hidden WM expects
`prompt_style=vla_policy`, `history=2`, `include_state=true`,
`rotate_images_180=true`, `obs_hidden_source=action_query`, and flattened
`[H, 1024]` embeddings.

The active action-hidden dataset configs set:

```text
dataset.expected_model_path
dataset.expected_encoder_state_ckpt
dataset.expected_time_horizon
dataset.expected_action_head_type
dataset.expected_obs_hidden_source
dataset.expected_prompt_style
dataset.expected_history
dataset.expected_include_state
dataset.expected_rotate_images_180
```

`LIBEROPixelRynnHiddenSequenceDataset` checks those against the sidecar
`preprocess_config.json` at startup, so stale hidden sidecars fail fast instead
of silently training.

## Output Layout

All configs should write under these roots:

```text
data/outputs/vla/
data/outputs/worldmodel/
data/outputs/dreamervla/
data/outputs/eval/
```

Do not create new top-level output categories unless the artifact type is
actually new. `data/` is intentionally ignored by git.

## Config Hygiene Rules

- Keep dataset names explicit: use `libero_goal`, `libero_10`,
  `libero_object`, or `libero_spatial`; avoid ambiguous tags.
- Keep VLA checkpoint, action-hidden sidecar, `action_head_type`, and
  `time_horizon` aligned.
- Put temporary experiments in `RUN_TAG`, not by copying another YAML.
- Prefer script environment variables for operational choices:
  `CUDA_VISIBLE_DEVICES`, `NUM_GPUS`, `BATCH_SIZE`, `NUM_WORKERS`,
  `RUN_TAG`, `OUT_DIR_BASE`.
- If a config is a new publishable baseline, add it to this file.
