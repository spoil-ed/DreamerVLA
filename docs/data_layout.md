# Data layout (`DVLA_DATA_ROOT`)

Single source of truth for everything DreamerVLA reads or writes outside the
repo. Code and data are connected by exactly one environment variable:

| Variable | Meaning | Default |
| --- | --- | --- |
| `DVLA_ROOT` | Project root, auto-derived by every formal script. | repo checkout dir |
| `DVLA_DATA_ROOT` | Data root; every path below lives under it. | `${DVLA_ROOT}/data` |

Migrating to a new machine means rsyncing `${DVLA_DATA_ROOT}`, cloning the repo,
and setting `DVLA_DATA_ROOT`. Provisioning from scratch means running the
Provenance commands below.

## Directory Tree

```text
${DVLA_DATA_ROOT}/
|-- ckpts/                          # pretrained weights
|   |-- VLA_model_256/<suite>/
|   |-- Action_World_model_512/<suite>/
|   |-- chameleon/tokenizer/
|   |-- models--Alpha-VLLM--Lumina-mGPT-7B-768/
|   |-- OpenVLA-OFT/<run>/
|   `-- Openvla-oft-SFT-traj1/<name>/
|-- dataset/
|   |-- libero/<suite>/             # raw LIBERO HDF5 demos
|   `-- calvin/                     # optional
|-- processed_data/                 # generated by preprocessing
|   |-- <suite>_marked_t_256/
|   |-- <suite>_no_noops_t_256/
|   |-- <suite>_no_noops_t_256_pi06_remaining_reward/
|   |-- <suite>_no_noops_t_256_pi0_legacy_action_hidden_vla_policy_h2/
|   |-- <suite>_image_state_action_t_256/
|   |-- convs/
|   |-- tokens/
|   |-- concate_tokens/
|   `-- <suite>_metainfo.json
|-- configs/<suite>/                # generated pretokenize dataset configs
|-- outputs/                        # training and eval runs
|-- logs/                           # preprocessing logs
|-- wheels/                         # cached install wheels
|-- tmp_ckpts/                      # staged eval checkpoints
`-- .libero/config.yaml             # generated LIBERO path config
```

Canonical raw LIBERO suites live at
`${DVLA_DATA_ROOT}/dataset/libero/<suite>`. Canonical generated datasets live at
`${DVLA_DATA_ROOT}/processed_data`.

## Asset Classes

### Pretrained Checkpoints: `ckpts/`

| Path | Content / format | Needed by | Provenance |
| --- | --- | --- | --- |
| `ckpts/VLA_model_256/<suite>/` | RynnVLA-002 VLA init, Hugging Face model files and safetensors. | VLA SFT, action-hidden sidecar, online RL. | `bash scripts/download_assets.sh` from `Alibaba-DAMO-Academy/RynnVLA-002`. |
| `ckpts/Action_World_model_512/<suite>/` | RynnVLA action world-model init. | Optional WM experiments. | `DOWNLOAD_ACTION_WM=1 bash scripts/download_assets.sh` (default on). |
| `ckpts/chameleon/tokenizer/` | `text_tokenizer.json`, `vqgan.yaml`, `vqgan.ckpt`. | Pretokenize, VLA SFT, sidecar generation. | `bash scripts/download_assets.sh` from `Alibaba-DAMO-Academy/WorldVLA`. |
| `ckpts/models--Alpha-VLLM--Lumina-mGPT-7B-768/` | Lumina-mGPT tokenizer and backbone files. | Pretokenize, VLA SFT. | `bash scripts/download_assets.sh` from `Alpha-VLLM/Lumina-mGPT-7B-768`. |
| `ckpts/OpenVLA-OFT/<run>/` | OFT SFT checkpoint directory, including model files, action head, proprio projector, and `dataset_statistics.json`. | OFT training and WM routes. | Produced by `CONFIG=openvla_oft_hdf5 bash scripts/train_vla.sh`. |
| `ckpts/Openvla-oft-SFT-traj1/<name>/` | One-trajectory OFT eval checkpoints. | Official-style OFT eval only. | `hf download Haozhan72/<repo> --local-dir ${DVLA_DATA_ROOT}/ckpts/Openvla-oft-SFT-traj1/<repo>`. |

### Raw Datasets: `dataset/`

| Path | Content / format | Needed by | Provenance |
| --- | --- | --- | --- |
| `dataset/libero/<suite>/*.hdf5` | Raw LIBERO demos, one file per task, HDF5 demos under `data/demo_*`. | All LIBERO preprocessing. | `bash scripts/download_assets.sh`; passes `--download-dir ${DVLA_DATA_ROOT}/dataset/libero` to the LIBERO downloader. |
| `dataset/calvin/` | CALVIN archives and optional extracted data. | Optional CALVIN experiments. | `DOWNLOAD_CALVIN=1 bash scripts/download_assets.sh`. |

### Preprocessed Products: `processed_data/`

All per-suite products are generated by
`TASK=<suite> bash scripts/preprocess/prepare_libero_data.sh`. Stage 4 delegates
to `scripts/preprocess/process_all_libero_data.sh`.

| Path | Content / format | Stage |
| --- | --- | --- |
| `<suite>_marked_t_256/` | Replayed demos with no-op marks; HDF5. | 1 |
| `<suite>_no_noops_t_256/` | Filtered training view; HDF5. | 2 |
| `<suite>_no_noops_t_256_pi06_remaining_reward/` | HDF5 with remaining-steps reward labels. | 3 |
| `<suite>_image_state_action_t_256/` | Extracted image, state, and action trees. | 4 |
| `convs/*.json` | Conversation JSON files for train, val_ind, and val_ood splits. | 4 |
| `tokens/<run>/files/*.pkl`, `tokens/<run>/record.json` | Pretokenized samples and records. | 4 |
| `concate_tokens/<suite>_his_1_*.json` | Concatenated token manifest. | 4 |
| `<suite>_no_noops_t_256_pi0_legacy_action_hidden_vla_policy_h2/` | Action-hidden sidecar matching `task.legacy_action_hidden.*`; token dim 1024, history 2, action-query source. | 5 |
| `<suite>_metainfo.json` | Task metainfo, including task language and init-state references. | 1 |
| `<suite>_no_noops_t_256_failures*` | Optional failure rollouts and sidecars for the latent classifier. | See `docs/classifier_revision_plan.md`. |

### Generated Configs: `configs/<suite>/`

`his_1_third_view_wrist_w_state_1_256_pretokenize{,_val_ind,_val_ood}.yaml`
files are written by preprocessing stage 5. They point at token manifests used
by `configs/task/<suite>.yaml`.

These generated YAML and JSON manifests can embed absolute paths. If
`${DVLA_DATA_ROOT}` changes after generation, rerun preprocessing stage 4-5 or
replace the old prefix inside `processed_data/{tokens,concate_tokens}/**/*.json`
and `configs/<suite>/*.yaml`.

### Outputs: `outputs/`

Training runs live under `outputs/<route>/<run>/checkpoints/`, with EMA copies
under `ema/` when `training.use_ema=true`. Eval runs live under
`outputs/eval/`. Old runs are safe to prune unless a config override such as
`init.world_model_state_ckpt` or `init.classifier_state_ckpt` points at them.

### LIBERO Path Config: `.libero/config.yaml`

Every formal script regenerates this file by default. It points LIBERO at code
assets under `${DVLA_ROOT}/third_party/LIBERO` and raw demos under
`${DVLA_DATA_ROOT}/dataset/libero`. Set `DREAMERVLA_WRITE_LIBERO_CONFIG=0` only
when intentionally preserving a manual config.

## Migration Checklist

1. Copy data: `rsync -a old:/path/to/dvla_data/ new:/path/to/dvla_data/`.
2. Clone the repo and install: `bash scripts/install_env.sh`.
3. Export the data root: `export DVLA_DATA_ROOT=/path/to/dvla_data`.
4. If pretokenize manifests moved across absolute paths, rerun preprocessing
   stage 4-5 or replace the old prefix in generated YAML / JSON manifests.
5. Verify config resolution with a Hydra compose check and ensure paths resolve
   under `${DVLA_DATA_ROOT}/ckpts`, `${DVLA_DATA_ROOT}/dataset/libero`, and
   `${DVLA_DATA_ROOT}/processed_data`.

## Existing-Machine Symlink Migration

If an older checkout already has raw LIBERO data under
`third_party/LIBERO/libero/datasets`, keep the data in place and create a
one-time symlink:

```bash
mkdir -p "${DVLA_DATA_ROOT:-data}/dataset"
ln -s "$(pwd)/third_party/LIBERO/libero/datasets" "${DVLA_DATA_ROOT:-data}/dataset/libero"
```
