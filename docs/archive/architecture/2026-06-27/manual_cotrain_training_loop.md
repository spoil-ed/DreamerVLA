# Manual Cotrain Training Loop

This note describes the manual-cotrain training loop at the module level. It is intended
as an editable architecture scaffold, not a line-by-line code explanation.

## Core Idea

Manual cotrain replaces real environment rollout with world-model rollout.

```text
VLA policy -> action chunk -> world model -> imagined observation/reward -> RL update
```

During training, the policy does not step in real LIBERO. Real LIBERO is only
needed for evaluation, data generation, or rebuilding the world model/reward
model.

## Main Modules

### Actor

The Actor is the policy being optimized.

In the current manual-cotrain setup, this is OpenVLA-OFT trained with GRPO/PPO-style
policy updates.

Responsibilities:

- Hold the trainable policy weights.
- Receive imagined trajectories.
- Compute advantages.
- Recompute current log probabilities.
- Apply the clipped policy loss.
- Produce updated weights for the next rollout round.

### Rollout Policy

The Rollout policy is an inference copy of the Actor.

Responsibilities:

- Receive observations from the world-model environment.
- Generate an action chunk.
- Return actions plus rollout-time metadata.

Typical returned data:

```text
actions
old_logprobs
forward_inputs
values, if enabled
policy version
```

The Rollout policy is periodically synchronized from the Actor.

### World Model Environment

The World Model Environment replaces the real simulator during training.

In manual cotrain this is WanEnv.

Responsibilities:

- Reset from an initialization dataset.
- Optionally use KIR, keyframe-initialized rollout.
- Maintain recent visual context.
- Consume action chunks.
- Generate imagined future frames.
- Produce the next observation.
- Produce reward using a reward model.
- Produce done/truncation signals.

Conceptually:

```text
current frames + task + action chunk
        |
        v
Wan world model
        |
        v
future frames + reward
```

### Trajectory Buffer

The Trajectory Buffer is the collected rollout data used for Actor training.

It stores the information needed to perform an on-policy update:

```text
observations / forward_inputs
actions
old_logprobs
rewards
dones
terminations
truncations
optional values
policy versions
```

The buffer is built from imagined interaction, not real LIBERO interaction.

### Evaluator

The Evaluator is separate from the training rollout.

For manual cotrain:

- training env: world model
- eval env: real LIBERO simulator

Evaluation is only triggered when explicitly enabled, for example by setting
`only_eval: True` or `val_check_interval > 0`.

## Training Loop

One training update can be viewed as the following sequence:

```text
1. Sync Actor weights to Rollout policy
2. Collect imagined rollouts using the world model
3. Build a trajectory batch
4. Compute advantages
5. Update Actor with PPO/GRPO loss
6. Save checkpoint and/or run evaluation if configured
```

Expanded one level:

```python
for update in range(num_updates):
    sync_actor_to_rollout()

    trajectories = []

    for rollout_epoch in range(num_rollout_epochs):
        obs = world_model_env.reset()

        for chunk_step in range(num_chunk_steps):
            rollout_output = rollout_policy.act(obs)

            next_obs, rewards, dones, infos = world_model_env.chunk_step(
                rollout_output.actions
            )

            trajectories.append({
                "forward_inputs": rollout_output.forward_inputs,
                "actions": rollout_output.actions,
                "old_logprobs": rollout_output.old_logprobs,
                "rewards": rewards,
                "dones": dones,
            })

            obs = next_obs

    batch = build_batch(trajectories)
    batch = apply_masks_and_filters(batch)
    batch["advantages"] = compute_advantages(batch)

    train_actor(batch)

    maybe_save_checkpoint()
    maybe_evaluate_on_real_env()
```

## Rollout Collection

Rollout collection is driven by the environment worker and rollout worker
together.

```text
WorldModelEnv.reset()
        |
        v
obs -> RolloutPolicy
        |
        v
action chunk -> WorldModelEnv.chunk_step()
        |
        v
next obs + reward + done
```

The important abstraction is that the policy acts on observations exactly as if
they came from an environment, but the transition is generated by Wan instead
of LIBERO.

## Actor Update

After rollout collection, the Actor receives a batch of imagined trajectories.

The update has three conceptual phases:

```text
trajectory batch
    -> advantage computation
    -> policy loss
    -> optimizer step
```

For GRPO, samples are grouped by the same initial condition or prompt. Rewards
inside the group are compared against each other to produce relative advantages.

For PPO-style loss, the Actor compares:

```text
current logprob
old rollout-time logprob
advantage
```

and applies a clipped objective.

## When Real Rollout Is Needed

Real rollout is not part of the normal manual-cotrain training loop.

It is needed in these cases:

- final or periodic evaluation in real LIBERO;
- generating data to train or rebuild the world model;
- training or validating a reward model;
- comparing against a real-simulator RL baseline;
- running a non-manual-cotrain config where the train environment is LIBERO.

## Practical Configuration Mapping

The main config ideas are:

```yaml
env:
  train:
    env_type: wan_wm
  eval:
    env_type: libero
```

This means:

```text
training rollout = Wan world model
evaluation rollout = real LIBERO
```

Parallelism is controlled by placement and environment count:

```yaml
cluster:
  component_placement:
    actor,env,rollout: all

env:
  train:
    total_num_envs: 64

algorithm:
  group_size: 8
  rollout_epoch: 16
```

With four GPUs and `actor,env,rollout: all`, each GPU hosts an Actor rank, a
Rollout rank, and an Env rank. Each Env rank owns one world-model copy and
multiple logical environments.

## Extension Points

Use this order when modifying the system:

1. Change reset behavior.
   Module: World Model Environment.

2. Change imagined transition behavior.
   Module: World Model Environment.

3. Change reward computation.
   Module: World Model Environment or Reward Model.

4. Change action sampling.
   Module: Rollout Policy.

5. Change advantage computation.
   Module: Actor / Algorithm.

6. Change policy loss.
   Module: Actor / Algorithm.

7. Change real-environment evaluation.
   Module: Evaluator.

## Minimal Mental Model

```text
Actor trains the policy.
Rollout uses a synced copy to sample actions.
Wan simulates the environment.
Reward model scores imagined futures.
GRPO/PPO turns imagined trajectories into policy updates.
Real LIBERO is only for evaluation or data/model construction.
```
