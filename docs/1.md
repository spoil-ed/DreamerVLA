## 1. 刚刚内容罗列

### TransDreamer 怎么学 dynamic

TransDreamer 不是靠 `real action vs shuffled action`。它主要靠：

```text
image reconstruction
+ reward prediction
+ KL(post, prior)
+ pcont / continue prediction
+ imagined actor-critic
```

其中核心是：

```text
prior = p(z_t | z_{<t}, a_{<t})
posterior = q(z_t | obs_t)
KL 让 prior 追 posterior
```

所以它学习 dynamics 的方式是：

```text
历史 latent + action → Transformer prior
真实 obs → posterior
prior 通过 KL 逼近 posterior
```

同时 TransDreamer 仍在 Dreamer 框架里训练 actor-critic，会用 `imagine_ahead` 在 imagined trajectory 上优化策略；你给的代码对比里也指出，TransDreamer 的 WM loss 是 `image + reward + kl + pcont`，并且 actor-critic 在同一训练流程中优化，而你们当前是两阶段分离。

## 2. 已有相关项目 / 方法

### A. ResWM

关键词：

```text
residual action
observation difference
action-conditioned change
```

它和你的 `delta_z` / `action-conditioned delta` 很接近：不是只预测绝对 next state，而是显式建模相邻观测差异和 action change。适合支撑：

```text
delta_latent_loss
normalized_delta_loss
action-conditioned change modeling
```

---

### B. World-Action Model, WAM

WAM 很贴近你现在的问题。它指出普通 world model 只预测 future observation，把 action 仅当 conditioning input；WAM 在 DreamerV2 里加入 **inverse dynamics objective**，从 latent state transition 预测 action，让 representation 捕捉 action-relevant structure。([arXiv][1])

对应你这里就是：

```text
(z_t, z_{t+1}) → a_t
```

它不是 shuffled action ranking，但目标非常相近：让 latent transition 不能忽略 action。

---

### C. Iso-Dream

Iso-Dream 也是 Dreamer 系列相关工作。它通过 inverse dynamics 学习，把 latent representation 按照对 action signal 的响应进行 factorization，从而区分 controllable / non-controllable dynamics。([NeurIPS Proceedings][2])

对应你这里：

```text
让 latent 区分 action-controllable dynamics
不要让背景/不可控变化主导 state
```

---

### D. Contrastive representation world model

比如 ReCoRe / Multi-View Dreaming 这类方向，核心不是直接重建，而是用 contrastive signal 增强 latent 的可分性。它们更接近：

```text
pred_future 接近 true_future
远离 wrong future / batch negatives
```

这和你现在的 action-ranking 有形式相似性。

## 3. real action vs shuffled action 相关方向

严格说，**直接在 world model 训练里写成：**

```text
real_action prediction closer to target than shuffled_action prediction
```

这种形式的论文不算主流标准配置；Dreamer / TransDreamer 本身都不是这么做的。

但有几个非常接近的方向：

### A. ENACT：shuffled action 用于 embodied world modeling evaluation

ENACT 不是训练方法，而是评测。它提出 inverse world modeling：给定有序视觉观测，模型要对 shuffled action sequence 重新排序；forward world modeling 中，给定初始观测和动作序列，模型要对 shuffled future observations 排序。([arXiv][3])

这和你的思想高度一致：

```text
如果模型真的理解 action → state transition
它应该能区分真实 action 顺序和 shuffled action
```

所以 ENACT 可作为你方法的 evaluation inspiration。

---

### B. LoCA Regret：shuffled action-index 用于测试 model-based RL 是否真正学 transition

LoCA 里有一个 shuffled action-index 设置，用来检验模型是否真的学到了有用 transition，而不是只靠表面相关性。它指出像 Dreamer / PETS 这类方法是模型和任务一起学习，而某些分阶段模型可能无法从 shuffled action-index 的预训练里学到有用 transition。([arXiv][4])

这支持你的判断：

```text
action identity 和 transition 必须绑定
否则 world model 对控制无用
```

---

### C. Batch RL Feature Analysis：shuffled action 用于因果/敏感性分析

这个工作用 learned world model 做 intervention，并比较 original action 和 shuffled action 对 state transition / reward 的影响。它不是训练 loss，但它直接使用了：

```text
a = original action
â = shuffled action
```

来分析 action 到 state / reward 的因果关系。([ACM数字图书馆][5])

这和你现在的诊断指标非常接近：

```text
real_to_target
shuffle_to_target
real_vs_shuffle
```

---

### D. WAM / inverse dynamics：不用 shuffled action，但解决同一个问题

WAM 不直接构造 shuffled negative，而是通过 inverse dynamics：

```text
z_t, z_{t+1} → a_t
```

来避免 action 被忽略。它的动机是：标准 world model 只预测未来 observation，action 只是条件输入，容易学不到 action-relevant structure；WAM 加 inverse dynamics objective 让 world model 必须理解动作如何驱动 transition。([arXiv][1])

这和你的 `real vs shuffled` ranking 是两种实现：

```text
你的方法：forward contrastive
WAM：inverse action prediction
```

目标都是：

```text
latent transition 必须包含 action 信息
```

---

## 4. 整合成你现在可用的研究表述

你可以这样总结你的方法来源：

```text
TransDreamer / Dreamer:
  主要依赖 KL + reconstruction + reward/pcont + imagination 学 dynamics；
  但没有显式 real-action vs wrong-action 对比。

WAM / Iso-Dream:
  说明 world model latent 需要 action-aware / controllable representation；
  inverse dynamics 是一种常见做法。

ENACT / shuffled-action evaluation:
  说明 shuffled action 是检验 embodied world model 是否理解 action-state causal relation 的有效方式。

我们的改法:
  把 shuffled-action 从 evaluation / analysis 变成 training negative；
  用 action-ranking loss 约束：
    dist(pred_real, target) + margin < dist(pred_shuffle, target)
  直接逼 WM 学 action-specific dynamics。
```

一句话：

> **你的 action-ranking 不是标准 Dreamer/TransDreamer loss，但它和 WAM 的 inverse dynamics、Iso-Dream 的 controllable dynamics、ENACT 的 shuffled-action world-model evaluation 是同一类思想：让 latent transition 必须携带 action 因果信息。**
